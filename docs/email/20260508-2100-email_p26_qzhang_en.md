PUBLICATION OF INTEREST
   Title:    "A prefetching indexing scheme for in-memory database systems"
   Authors:  Qian Zhang, Haoyun Song, Kaiyan Zhou, Jianhao Wei, Chuqiao Xiao
   Venue:    Future Generation Computer Systems (FGCS), Volume 156,
             Elsevier, 2024
   DOI:      10.1016/j.future.2024.02.018

Your path prefetcher with jump-pointer mechanics in the above-mentioned work
is a key comparison point for one of our main research gaps: hot-path-based
cache adaptation via probability hints in trie-based index structures. Your
reported 1.7x speedup over the prior state of the art is exactly the kind
of effect we want to study under our cache-engine framework across many
algorithm building blocks.

Question: Is the original code (path prefetcher + jump-pointer prefetcher
for versioned tuples) available for academic diploma-thesis use?

We follow a directive from Prof. Habich requiring exact-copy integration of
original code via an adapter pattern, so that your implementation can be
compared authentically in a building-block cross-permutation study across
roughly 30 published index algorithms.

If a direct code release is difficult, even pseudocode excerpts from your
internal notes would be sufficient for a clean re-implementation. We will
cite your work appropriately and preserve any license you choose for the
code. The code is treated as separate from the FGCS paper itself — your
choice for the code license (e.g. MIT or Apache-2.0) is independent from
the Elsevier publication terms.

I have addressed this email to Ms. Zhang as the lead author, and to
Prof. Gong as the institutional senior contact at the Software Engineering
